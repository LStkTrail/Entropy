# 学习笔记：Vibe Coding 初探

**核心主题**：通过游戏与工具开发，验证 AI 辅助编程的实战能力与“心流”体验。

## 1. 实践项目复盘

* **贪吃蛇 (Snake Game)**：
* *目标*：测试 AI 对经典游戏逻辑（状态管理、碰撞检测）及 Canvas 渲染的生成能力。
* *结果*：AI 能够迅速构建完整的游戏循环，只需通过自然语言调整“速度”、“颜色”或“增加无敌模式”等需求，无需手写底层逻辑。


* **Chrome 翻译插件**：
* *目标*：验证 AI 处理特定框架规范（Manifest V3）、DOM 操作及外部 API 调用的能力。
* *结果*：跳过了查阅繁琐文档（如 Chrome Extension 文档）的过程，直接生成可运行的脚手架，从“查文档-写代码”转变为“提需求-调试-优化”。

## 2. Vibe Coding 核心方法论

* **交互模式**：不再是逐行编写，而是“Prompt -> Code -> Review -> Refine”的迭代循环。
* **角色转变**：从**Coder**（关注语法、API细节）转变为**Architect/PM**（关注功能设计、用户体验、逻辑完备性）。
* **效率突破**：极大地降低了“上下文切换”的成本，让思维始终停留在解决问题的层面，而非陷入语法细节的泥潭。

## 3. 总结

Vibe Coding 不仅仅是代码补全，更是一种**“所想即所得”的开发范式**。在处理中小型项目时，它能让开发者进入一种极高效率的“心流”状态（Vibe），将原本数小时的开发周期压缩至分钟级，且保持了代码的可用性与逻辑连贯性。
