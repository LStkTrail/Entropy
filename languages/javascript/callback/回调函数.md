# 回调函数

---

## 1 核心定义与本质

### 1.1 什么是回调函数？

字面理解：Call Back = 回头调用

> **什么是回调函数？**
>
> 吧函数 A 当作参数传递给函数 B，当函数 B 执行到某个时刻（或者完成任务后），它会执行函数 A。此时，函数 A 就是函数 B 的回调函数。

### 1.2 代码解析

```js
function B(msg, A) {
  console.log(`This msg is : ${msg}`)
  
  // B Callback A
  A();
}

B("Hello", () => console.log("Mamba Out"))
```

## 2 回调函数分类

> **回调并不等于异步**

### 2.1 同步回调

- **行为**：立即执行，完全阻塞后续代码
- **示例**：数组遍历（`forEach`，`map`，`filter`）
- **特征**：阻塞后续代码，回调不完成，主程序无法继续，后续代码无法执行

```js
const arr = [1, 2, 3];
arr.forEach(function(num) { // 这个函数就是同步回调
  console.log(num); 
});
console.log("结束"); 
// 顺序：1 -> 2 -> 3 -> 结束（绝对保证顺序）
```

### 2.2 异步回调

- **行为**：不会立即执行，而是进入到任务队列，在稍后执行
- **示例**：`setTimeout`, `addEventListener`, Ajax 请求, Node.js 读取文件
- **特征**：不会阻塞后续代码，后续代码先执行，等到异步动作完成后，执行回调

```js
setTimeout(function() { // 这个函数是异步回调
    console.log("时间到了");
}, 1000);
console.log("结束");
// 顺序：结束 -> (等1秒) -> 时间到了
```

## 3 注意事项与难点

### 3.1 `this` 指向丢失问题

当把一个对象的方法当作回调传出去时，`this` 往往会变成全局对象或者 `undefined`

**错误场景：**

```js
const user = {
  name: "Kobe",
  sayHi: function() {
    console.log("Mamba out!")  
  }
}

// 输出为 undefined，因为 setTimeout 中回调执行时，this 不再指向 user，而是该函数本身
setTimeout(user.sayHi, 1000)
```

**解决方案：**

1. 箭头函数：因为箭头函数没有自己的 `this`，所以会继承外层的 `this`

```js
setTimeout(() => user.sayHi(), 1000); // 外层的 this 即为 user 的 this
```

2. `bind` 绑定：强制锁定 `this`

```js
setTimeout(user.sayHi.bind(user), 1000);
```

### 3.2 回调地狱

- **定义**：多层嵌套的回调函数
- **本质**：多层以来，控制权的转移
- **解决方法**：Promise，Async/Await

### 3.3 Inversion of Control

你的 A 是一个好 A，但是托付的 B 是一个坏 B，就会面临风险（例如将回调交给一个第三方库）：

- 可能调用你的回调 0 次或 100 次
- 可能吞掉你的报错

## 4 高级设计模式

### 4.1 错误优先回调（Error-First Callback）

**规则**：回调的第一个参数必须保留给错误对象 `err`

```js
function divide(a, b, callback) {
  setTimeout(() => {
    if (b == 0) callback(new Error("除数不能为 0"), null)
    else callback(null, a / b)
  }, 1000)
}

divide(10, 2, (err, result) => {
  if (err) return console.error(err.message)
  console.log("结果是：", result)
})

divide(10, 0, (err, result) => {
  if (err) return console.error(err.message)
  console.log("结果是：", result)
})
```

## 5 关键知识图谱

| 知识点 | 核心内容 | 备注 |
|---|---|---|
| 基础语法 | 函数作为参数传递 | 注意不要带括号，因为传递的是函数的引用，而不是函数的执行结果 |
| 执行时机 | Sync V.s. Async | 同步场景（ `forEach` 等）；异步场景（ `setTimeout` 等） |
| 作用域 | 闭包 | 回调函数天然能够访问外部变量（因为它就是闭包 |
| 上下文 | `this` 指向 | 修正 `this` 的方式 - 箭头函数 / `bind` |
| 参数约定 | Error-First | 为了保证能够处理所有错误情况 |
| 控制流 | 回调地狱 | 回调地狱会导致代码难以维护，同时控制权会转移 |
| 控制反转 | 控制权的信任问题 | Promise 比 Callback 更加安全 |
| API 设计 | 一致性（Zalgo） | API 必须保持始终同步或始终异步，不可以混用 |

---

## 6 总结

回调函数是 JavaScript 的**原子**。虽然可以使用 Promise 和 `async/await`，但是底层依旧建立在回调机制上（微任务队列）。
